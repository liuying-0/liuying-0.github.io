图的应用

发表于2023-07-29|更新于2023-07-29|408数据结构图的应用
|字数总计:3.4k|阅读时长:11分钟|阅读量:

图的应用主要包括：最小生成树、最短路径、拓扑排序和关键路径，考察形式主要以结合图的实例来考察算法的具体操作过程，需学会手工模拟给定图的各个算法的执行过程，掌握对给定模型简历相应的图取解决问题的方法。

最小生成树

图的最小生成树是指在一个连通图中找到一棵包含所有顶点的子树，使得这棵子树的边的权重之和最小，考纲涉及到的Prim算法和Kruskal算法主要基于贪心算法，其最主要的区别在于Prim算法以一个点为起始，利用贪心算法，逐渐将各个点连接入生成树中；而Kruskal算法则是以边为起始，不断选最权重最小且不构成环的边，最终实现最小生成树。

Prim算法

  • Prim算法是一种贪心算法，从一个起始顶点开始，逐步扩展生成最小生成树。

  • 首先选择一个起始顶点，将它加入最小生成树，并将其与其它未加入最小生成树的顶点之间的边的权重添加到一个优先级队列中。

  • 然后，从优先级队列中选择权重最小的边对应的顶点，将该顶点加入最小生成树，并更新优先级队列中的边的权重。

  • 重复上述步骤，直到最小生成树包含所有顶点为止。

Kruskal算法

  • Kruskal算法也是一种贪心算法，它首先将所有边按照权重从小到大进行排序。

  • 可以使用并查集数据结构来判断是否会形成回路。

最短路径

最短路径问题是图论中一个经典的问题，即在给定带权重的图中找到两个顶点之间的最短路径，使得路径上的边权重之和最小。主要有Dijkstra算法和Floyd算法。其中Dijkstra算法主要用于求单源最短路径，即某一顶点到其他各顶点的最短路径；Floyd算法则是用于求各个顶点之间最短路径问题，即每对顶点间的最短路径。

Dijkstra算法

Dijkstra算法，其核心思想是从一点出发，遍历所有与之相连的所有点，找到距离最短的那个，加入已选中的集合中，表示已找到到该点的最短路径，随后更新距离表，寻找从源点出发，到达其他未到达点的最短路径。

  1. 初始化：
    • 创建一个距离数组dist[]，用于存储从源点到各个顶点的最短距离。初始时，将dist数组的所有元素设置为无穷大（表示未知距离），然后将源点的dist值设为0，表示源点到自身的距离为0。
    • 创建一个前驱结点数组path[]，用于记录从源点到各个顶点的最短路径的前驱结点。初始时，将path数组的所有元素设置为一个特定的值（例如-1），表示还没有找到路径的前驱结点。
    • 创建一个访问标记数组visited[]，用于标记节点是否已经被访问。初始时，将所有节点的访问标记设为false。
  2. 选择源点：
    • 选择一个源点作为起始节点，并将该节点设为当前节点。
  3. 更新距离数组：
    • 从当前节点开始，遍历当前节点的所有相邻节点。
    • 对于每个相邻节点，如果通过当前节点到达该相邻节点的路径比原来的路径更短，则更新距离数组中该节点的距离值，并将当前节点记录在path数组中作为该相邻节点的前驱结点。
  4. 标记当前节点：
    • 在更新距离数组后，将当前节点标记为已访问。
  5. 选择下一个节点：
    • 从未标记的节点中选择一个dist数组中距离最小的节点作为下一个当前节点，重复步骤3和4，直到所有节点都被标记为已访问，或者没有可达路径为止。
  6. 输出结果：
    • 最后，距离数组dist[]中存储的即为从源点到其他所有节点的最短距离，而前驱结点数组path[]中存储的是从源点到各个顶点的最短路径上的前驱结点信息。
  7. 还原最短路径：
    • 根据path[]数组的信息，我们可以还原源点到各个顶点的最短路径。从目标顶点开始，通过反向遍历path[]数组，即可找到源点到目标顶点的完整最短路径。

Floyd算法

Floyd算法是一种用于求解图中任意两个顶点之间最短路径的算法，也称为全源最短路径算法。Floyd算法采用动态规划的思想，通过逐步地考虑图中不同的顶点作为中间节点，来逐步更新任意两个顶点之间的最短路径。

  1. 初始化：
    • 创建一个二维数组dist[][]，用于存储任意两个顶点之间的最短距离。初始时，将dist[i][j]设置为边的权重（如果顶点i和j之间有边相连），否则设置为无穷大。
    • 对角线元素dist[i][i]设为0，表示顶点到自身的距离为0。
  2. 动态规划更新：
    • 从0号顶点到V-1号顶点（V为图的顶点数）依次考虑每个顶点作为中间节点，进行动态规划更新。
    • 对于每个顶点k（从0到V-1），遍历所有的顶点i和j，尝试通过顶点k缩短从顶点i到顶点j的距离：
      • 如果dist[i][j] ＞ dist[i][k] + dist[k][j]，则更新dist[i][j]为dist[i][k] + dist[k][j]，即通过顶点k可以得到更短的路径。
  3. 输出结果：
    • 在动态规划更新完毕后，dist[i][j]中存储的即为从顶点i到顶点j的最短距离。

拓扑排序

AOV网

AOV网（Activity On Vertex network）是一种用于描述工程项目或任务的有向无环图（DAG，Directed Acyclic Graph）。在AOV网中，顶点表示活动（任务或工程项目），有向边表示活动之间的依赖关系，且不存在环路，即没有循环依赖。

拓扑排序

由一个有向无环图的顶点组曾的序列，当且仅当满足下列条件时，称为该图的一个拓扑排序：

  • 每个顶点出现且仅出现一次
  • 若顶点A在序列中排在顶点B的前面，则在图中不存在从顶点B到顶点A的路径

对AOV网进行拓扑排序

  1. 选择入度为0的顶点：在AOV网中，找到所有入度为0的顶点（即没有依赖其他顶点的顶点），将它们作为起始节点。
  2. 输出起始节点：将入度为0的顶点输出，并从AOV网中删除这些顶点及它们的出边，表示这些顶点已经被处理。
  3. 更新入度：更新剩余顶点的入度，即将与刚刚输出的顶点相邻的顶点的入度减1。
  4. 重复过程：重复步骤1~3，直到所有顶点都被输出。

关键路径

AOE网

AOE网（Activity On Edge network）是一种用于描述工程项目或任务的有向带权图，也称为带权有向图。在AOE网中，顶点表示活动（任务或工程项目），有向边表示活动之间的依赖关系，且每条边都带有权重（也称为活动时间或持续时间）表示活动的执行时间。

关键路径

关键路径是指在AOE网（或带权有向图）中，从起始节点到终止节点的所有路径中，总持续时间最长的路径。关键路径上的活动对整个工程项目的进度具有决定性的影响，即如果关键路径上的任何一个活动延迟了，整个项目的完成时间都会延迟。

求关键路径的方法

  1. 计算最早开始时间（ES）和最早完成时间（EF）：
    • 从起始节点开始，按照拓扑排序的顺序计算每个活动的最早开始时间（ES）和最早完成时间（EF）。
    • 对于每个活动，最早开始时间（ES）为其前驱活动的最早完成时间（EF）中的最大值。
    • 最早完成时间（EF）等于最早开始时间（ES）加上该活动的持续时间。
  2. 计算最晚开始时间（LS）和最晚完成时间（LF）：
    • 从终止节点开始，逆向计算每个活动的最晚开始时间（LS）和最晚完成时间（LF）。
    • 对于每个活动，最晚完成时间（LF）为其后继活动的最晚开始时间（LS）中的最小值。
    • 最晚开始时间（LS）等于最晚完成时间（LF）减去该活动的持续时间。
  3. 计算活动的总浮动时间（Total Float，TF）：
    • 活动的总浮动时间（TF）表示该活动可以延迟的时间而不影响整个项目的总持续时间。
    • 活动的总浮动时间（TF）等于最晚开始时间（LS）减去最早开始时间（ES）或最晚完成时间（LF）减去最早完成时间（EF），二者等价。
  4. 找出关键路径：
    • 在完成上述计算后，找出所有活动的总浮动时间（TF）为0的路径，这些路径即为关键路径。
    • 关键路径上的活动没有浮动时间，必须按照最早时间完成，否则整个项目的完成时间将被延迟。

（注：ve为事件最早开始时间，vl为事件最晚开始时间，e为活动最早开始时间，l为活动最晚开始时间）
